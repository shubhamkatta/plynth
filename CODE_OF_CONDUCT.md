# Code of Conduct

Plynth adopts the **Contributor Covenant v2.1** verbatim as its code of
conduct. The full text is at:

<https://www.contributor-covenant.org/version/2/1/code_of_conduct/>

It applies to all project spaces: this repository (issues, pull requests,
discussions, code review), any official chat channels, and project events.

## Reporting

Report concerns privately to **shubhamkatta7@gmail.com**. Reports are
read by the project owner only and handled per the Covenant's
"Enforcement Guidelines" section. We'll acknowledge within 72 hours and
work toward a resolution within 30 days.

If you'd prefer to report via GitHub, use a
[private security advisory](https://github.com/shubhamkatta/plynth/security/advisories/new)
— it routes the same place but is auditable.

## Scope summary

- **Be respectful, patient, and constructive** — assume good intent;
  give specific, actionable feedback.
- **Welcome newcomers** — every maintainer was a first-time contributor
  once. Onboarding questions are not "noise."
- **No harassment, personal attacks, or discriminatory language** — not
  in issues, PRs, commits, code comments, or anywhere else.
- **Respect maintainer time** — search before asking; minimal reproducers
  for bugs; align on design before coding for large features.

Maintainers may, at their discretion, remove comments, close issues,
revert commits, or block accounts that violate these standards.
